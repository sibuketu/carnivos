# 用語「実装」の定義と表記一覧

> RULES.md 2.3 に基づく。「実装」は**これ以上することがない状態**になってからだけ使う。

---

## 表記ルール

| 状態 | 使う表現 | 使わない表現 |
|------|----------|--------------|
| 機能が仕様どおり動き追加作業が不要 | 実装済み / 実装した | — |
| 導線・画面だけ、API未接続 | 対応済み / 導線のみ / 画面まで | 実装済み |
| 一部だけ動いている | 部分対応 / 一部対応 | 実装済み |
| まだやっていない | 未対応 / 未実装 | — |

---

## 見直し済み（主要）

- `docs/決定の背景_2026-02-03.md`: デバイス連携を「導線のみ対応済み」「API連携は未対応」に表記統一。
- その他ドキュメントの「実装済み」は、上記ルールに照らして必要なら「対応済み」「完了」等に随時修正する。

---

## トッピング（ButcherSelect）

- **内容**: 肉にのせる追加分（脂・タロー・Mg・Na）。塩は別で `saltGrinds` で入力済み。
- **方針**: トッピング専用機能は作らない。タロー・バター等は「部位選択で選んで量を入れる」通常の追加フローでよい（一括で同じ扱い）。`toppingNutrients` は 0 固定のまま運用。
