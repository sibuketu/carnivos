## なぜこのアプリを作ったか（ミッション・根拠）

- **アプリ内では紹介しない**: この「なぜ」はアプリ内の画面では紹介しない。Obsidian には余計なくらい書いてよい。SNS・広告・YouTube のネタに使う場合は、ここから再編集して使う。
- **問題意識**: 不健康な食品を食べること自体は moral に問題ではない。しかし「健康になりたい」と思って、主流の栄養情報（糖質制限・低脂質・食品ピラミッド等）に従い、不健康な食べ方をして不健康になるのは、**誤情報に基づく結果**であり、最も避けたいことだと考えている。
- **根拠のレベル**: 既存の栄養学やガイドライン（USDA、食品ピラミッド、多くの病院・メディアのアドバイス）への不信感は、単なる感情ではない。Carnivore Doctors（Ken Berry, Shawn Baker 等）や生化学・論文（PubMed、Cronometer 等のデータ）に触れ、「データは実在するが解釈が政治・商業・古い科学に引きずられている」と判断した上での不信感である。つまり「正しい情報」を「手軽に」届けたい。
- **アプリの役割**: 世界一の Carnivore アプリを作ることで、正しい情報に基づく選択がしやすくなり、誤情報による不健康を減らす一助にしたい。**Obsidian のメモは簡潔にしなくてよい。根拠・背景は余計なぐらい書いてよい。** その中から SNS・広告・YouTube 用に再編集して使う。

## 明確に記録できているもの（根拠・背景）

### ログインとサブスクの順番
- **結論**: オンボーディングの**前**にサブスク（価格）を見せる。**初回起動（未ログイン）**の流れは **起動 → サブスク画面（またはカルーセル→サブスク）→ ログイン/登録 → オンボーディング**。
- **「いきなりサブスク」が適用されるのは初回起動だけ**: **別のスマホで既存アカウントにログインするとき**は、いきなりサブスクを出さない。**ログイン/新規作成を先に表示**し、ログイン後にホーム（またはサブスク状態に応じた画面）へ。既存ユーザーの別デバイスログインは「ログイン → ホーム」が自然。
- **ベンチマーク**: 他アプリがどうしているかの調査・比較。ログインとサブスクの順番の他アプリベンチマークは別途調査するとよい。
- **データ**: オンボーディングの**前**にペイウォールを見せると trial 登録率が 5〜6 倍良い、というデータに合わせている。

### 価格を先に見せる（upfront paywall）
- **何に対して**: オンボーディングの**後**にペイウォールを出す場合と比べて、オンボーディングの**前**に出すと trial 登録率が 5〜6 倍良い（RevenueCat 等）。
- **開発日記用メモ**: この事実は開発の経緯・戦略を世に公開するときに使えるのでメモする。

### ペルソナ的に起動時いきなりサブスク
- **結論**: いきなりサブスクでよい。離脱率低減のメリットを優先。
- **オンボーディング**: 移行スタイル（始めたばかり / 移行中 / 適応済み）はユーザーに選ばせる。サブスク期間はデフォルト中間、設定で後から変更可能。

### いきなり派 vs 徐々に派
- **決定**: どちらか一方にはしない。ユーザーに選ばせる。3 選択肢（始めたばかり / 移行中 / 適応済み）は採用。
- **期間の目安**: 徐々に＝3〜6 ヶ月、いきなり＝1〜2 週間〜1 ヶ月。

### オンボーディングで何を表示するか
- **最低限がいい**: 離脱率を下げるため、表示項目は必要最小限。進捗表示（1/5 等）を反映。

### 「共有しませんか」モーダル
- **調査結果**: 公的データは見つからず。強制感があるのでほとんど意味ないと推測。モーダルは出さず、シェアボタンだけ置く。

### 共有ボタン（SNS リンク）
- 各種 SNS のリンクを出す案を採用。YouTube のような矢印で一覧が出る UI を既存アプリから調査して採用。

### 体重計連携（タニタ等）
- タニタに限らず対応。体脂肪入力の日記のところに連携ボタン。タニタで体重登録時は体脂肪も登録済みに（再連携不要）。
- **現状（用語ルールに合わせて明記）**: 日記の「デバイス連携」ボタンは健康デバイス画面への**導線のみ**対応済み。HealthDevice 画面では手動入力と Google Fit 取得まで対応。体重計・体脂肪計の**直接 API 連携（Withings / タニタ / Health Connect 等）は未対応**。「実装」は「これ以上することがない」状態になってからだけ使う。

### 3 モード + カスタム
- シンプル / スタンダード / ディテール + **カスタム**。シンプルで「もっと見る」→スタンダード→ディテール。シンプル＝電解質と脂質のみ（タンパク質は表示しない）。

### カルマゲージ（旧 Meat Gauge）
- アプリ内の機能名は必ず「カルマゲージ」。

### 牛の部位選択（シルエット）
- シルエットは採用しない。部位はリスト（文章）で選ぶ。

### 機能紹介の形式
- 軽い説明→タップで詳細。**詳細をタップしたときに実際の UI を見せる**。伝達の確実性と離脱抑制のバランスのため。

### Detective AI の目標値変更
- 「変更を推奨します。変更しますか？」＋変更ボタン。

### Prompt Chips
- 「この機能であなたの体調を〜」からプロンプトを選ぶ形式で OK。

### アップデートの曜日
- **毎週月曜**にリリース。

### 監視対象
- 名前は「監視対象」のまま。監視対象＝改善候補でもある。
