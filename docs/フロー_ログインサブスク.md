# ログイン・サブスクのフロー仕様

> 決定の背景に合わせた目標フロー。現状の実装とは異なる部分あり。実装時はこのファイルを正とする。

---

## 初回起動（未ログイン・初回インストール）

1. **起動**
2. **Consent**（必要なら）
3. **サブスク画面**（価格を見せる。カルーセル→サブスクでも可）
4. **ログイン / 新規登録**
5. **オンボーディング**（プロフィール等・最小限）
6. ホーム

※ データ: オンボーディングの**前**にペイウォールを見せると trial 登録率が 5〜6 倍良い、という根拠。

---

## 既存ユーザーが別デバイスでログインする場合

1. **起動**
2. **ログイン / 新規作成を先に表示**（いきなりサブスクは出さない）
3. ログイン後 → **ホーム**（またはサブスク状態に応じた画面）

---

## 現状の実装との差

- **現状**: consent → onboarding → （オンボーディング内の最後で支払いモーダル）。サブスクを「オンボーディングの前」に出していない。別デバイスでのログイン時も「初回フロー未完了」でない限り auth に飛ぶが、「サブスクを先に出さない」の明示分岐はなし。
- **目標**: 上記の通り。初回はサブスク→ログイン→オンボーディング。既存ユーザー別デバイスはログインを先に。
