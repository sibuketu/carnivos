# UI確認ガイド（画像なしで判断）

## 画像を送らなくても確認できる方法

### 1. ブラウザの開発者ツールで確認

**F12キーを押して開発者ツールを開く**

#### Consoleタブ
- **エラーメッセージをコピーして送る**
  - 赤いエラーがあれば、そのメッセージをそのまま送ってください
  - 例: `Uncaught TypeError: Cannot read property 'map' of undefined`

#### Networkタブ
- **「Disable cache」にチェックを入れる**
- **ページをリロード（Ctrl+Shift+R）**
- **失敗しているリクエスト（赤い行）があれば、そのURLを送る**

#### Elementsタブ
- **問題のある要素を右クリック → Inspect**
- **HTMLの構造を確認**
  - 例: 「期間フィルターボタンが見えない」→ Elementsタブで `.history-period-filter` を検索して、存在するか確認

### 2. 具体的な問題を文字で説明

画像の代わりに、以下の情報を送ってください：

#### 例1: HistoryScreen
```
- 期間フィルターボタン（Today, 7 day, 30 day, All, ⭐ All）が表示されない
- タブ（Summary, History, Photo Gallery）が表示されない
- ログが表示されない
- エラーメッセージ: [ここにエラーを貼り付け]
```

#### 例2: HomeScreen
```
- 栄養ゲージが表示されない
- 「Nutrient Breakdown」セクションが見えない
- 「Show More」ボタンしか表示されない
- エラーメッセージ: [ここにエラーを貼り付け]
```

#### 例3: ProfileScreen
```
- タイトルが「Setting」になっていない（「Settings」と表示される）
- 言語設定ボタンが表示されない
- エラーメッセージ: [ここにエラーを貼り付け]
```

### 3. スクリーンショットの代わりに使える情報

1. **ブラウザのURL**: `http://localhost:5174` など
2. **表示されている画面**: History / Home / Profile / Labs
3. **具体的な問題**: 「ボタンが見えない」「文字が化けている」「エラーが出る」など
4. **Consoleのエラー**: 開発者ツールのConsoleタブに表示されるエラーをコピー

### 4. よくある問題と確認方法

#### 問題: 何も表示されない
- **確認**: Consoleタブでエラーを確認
- **送る内容**: エラーメッセージ全体

#### 問題: 一部の要素が表示されない
- **確認**: Elementsタブで該当要素を検索
- **送る内容**: 「`.history-period-filter` というクラス名の要素が見つからない」など

#### 問題: スタイルがおかしい
- **確認**: Elementsタブで該当要素を選択 → Stylesパネルを確認
- **送る内容**: 「`.history-period-btn` の `display` が `none` になっている」など

---

## 使い方

1. **ブラウザで `http://localhost:5174` (または5175) を開く**
2. **F12で開発者ツールを開く**
3. **問題を確認**
4. **上記の情報をチャットに送る**

画像を送る必要はありません！
