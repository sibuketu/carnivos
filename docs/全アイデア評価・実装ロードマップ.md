## 2. 最優先で実装すべきTop 10（v1.0リリース前必須）

### 🔴 P0: v1.0リリース前に必須（8件）

| 順位 | ID | アイデア名 | 理由 | 実装時間 |

| 2 | A2 | ButcherSelect動的目標値対応 | A1の依存機能。追加前後で目標値が一致しないとUX破綻。 | 1時間 |
| 3 | B1 | 水分管理機能 | カーニボアでは水分管理が電解質バランスに直結。必須機能。 | 4時間 |
| 4 | B2 | 電解質バランス表示 | 水分管理と密接に関連。カーニボア特有の価値提供。 | 2時間 |
| 5 | C1 | 写真解析フォローアップクエスチョン | 写真解析の精度向上に不可欠。analyzeFoodNameには既にあるが、analyzeFoodImageにはない。 | 3時間 |
| 6 | C4 | バーコードモバイル対応 | 現状「このブラウザは対応してません」で機能が使えない。モバイルアプリなのに致命的。 | 1日 |
| 7 | H1 | 断食タイマー通知実装 | 機能は実装済みだが通知なしでは実用性が低い。 | 4時間 |

## 15. 実装判断フローチャート

```
新しいアイデアが出たら
  ↓
【Step 1】妥当性評価
  ├─ Excellent: 必須機能 → Step 2へ
  ├─ Good: 推奨機能 → Step 2へ
  ├─ Fair: あれば良い → Step 2へ（優先度低）
  ├─ Questionable: 再検討 → 仕様明確化後に再評価
  └─ Not Recommended: 実装しない → 終了

【Step 2】ユーザー価値評価
  ├─ Critical: なければ困る → v1.0（P0）
  ├─ High: 大きな価値 → v1.1（P1）
  ├─ Medium: 便利 → v1.2-1.5（P2）
  └─ Low: nice to have → v2.0+（P3-P4）

【Step 3】技術リスク評価
  ├─ High Risk → 実装計画慎重に。MVP優先。フォールバック用意。
  ├─ Medium Risk → 注意して実装。ユニットテスト充実。
  └─ Low Risk → 通常通り実装。

【Step 4】依存関係チェック
  ├─ 依存する機能が未実装 → 依存機能を先に実装
  └─ 依存関係なし or 実装済み → 実装開始

【Step 5】リソース配分
  ├─ 実装時間 < 1日 → 即実装
  ├─ 実装時間 1-3日 → 週次スプリントに組み込み
  ├─ 実装時間 1-2週間 → 月次スプリントに組み込み
  └─ 実装時間 > 2週間 → フェーズ分割して段階的実装
```
